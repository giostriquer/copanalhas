# Documentation

This docs folder is the source of truth for Copanalhas until implementation code
exists. Keep changes small and tied to a concrete product or safety decision.

## Map

- `product-brief.md`: purpose, users, and scope.
- `architecture.md`: initial component boundaries.
- `discord-ingestion.md`: Discord API access and data minimization rules.
- `data-sources.md`: World Cup match data source policy.
- `scoring-rules.md`: prediction and leaderboard rules.
- `security-privacy.md`: secrets, privacy, and abuse boundaries.
- `testing-policy.md`: initial test strategy.
- `conventions/implementation-patterns.md`: reviewable implementation patterns.
